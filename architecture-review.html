#1Strong
BigScreen.vue

BigScreen.vue — 超级组件需拆分

🚫 问题

737 行 SFC：模式切换 + 数据获取 + 8个子组件状态 + 图表ref + 回放控制 + 详情弹窗。修改数据获取可能意外影响弹窗——违背了 Locality 原则。

✅ 方案

拆为 DashboardShell（布局壳）+ useDashboardState（composable）+ DetailPanel（独立组件）。

#2Strong
ConvergeAnalysis.vue

ConvergeAnalysis.vue — 1200+ 行地理分析巨兽

🚫 问题

地图初始化 + 网格分析(Turf.js) + 评分算法 + 5个环境图层 + 工具栏 + 信息弹窗 + 选址模拟 + CSS 全挤在一个文件。修改评分公式需理解上千行上下文。

#3Strong
Cards.vue / BigScreen.vue

Cards.vue — Props 接口不匹配（运行时 Bug）

🚫 问题

BigScreen 传 :orders="orders"（Array），但 Cards 声明 ordersCount: Number。卡片显示的「需求单量」永远是 0。

✅ 方案

统一接口：Cards 接收 orders: Array，内部 computed 计算 count。

⚠️ 当前： BigScreen template: :orders="orders" → Cards props: ordersCount: Number → Vue 不报错但值永远 undefined
#4Strong
MapContainer.vue

MapContainer.vue — 含幽灵函数与死代码

🚫 问题

第159-163行 stopPlayback() 引用未声明的 timer、isPlaying——这个函数永远不会被调用，且执行就崩溃。另有重复的热力图初始化代码。

✅ 方案

直接删除 第159-164行。合并重复的热力图初始化。

🚫 问题

#6Worth Exploring
数据层

数据字段命名不一致 — 缺统一 Adapter

🚫 问题

BigScreen 第262行映射了 rName || r_name、rLat || r_lat || rlat 等变体。MapContainer第94行又做了一遍。数据清洗逻辑散落两处。

✅ 方案

创建 adapters/serviceOrderAdapter.js 统一规范化，所有组件通过Adapter消费。

AMap API Key 硬编码

🚫 问题

#8Worth Exploring
BigScreen.vue

缺少加载/错误/空状态处理

🚫 问题

fetchAll() 失败只有 console.error，用户看到空白页。接口不完整——调用者不知道模块可能有 loading/error/empty 三种状态。

✅ 方案

引入 { loading, error, data } 三态模式，loading 显示骨架屏，error 显示重试按钮。
